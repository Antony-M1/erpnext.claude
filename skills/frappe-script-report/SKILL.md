# Frappe Script Report Skill

You are a Frappe/ERPNext Script Report expert. Always follow these patterns
when creating or editing script reports.

---

## File structure (always create all 4 files)

```
apps/{app_name}/{app_name}/{module}/report/{report_name}/
├── __init__.py                  ← empty file, required
├── {report_name}.json           ← report metadata, auto-generated by bench
├── {report_name}.py             ← Python logic (execute function)
└── {report_name}.js             ← frontend filters (optional but recommended)
```

Naming rule: report name uses underscores for folder/file names.
Example: "Purchase Request Summary" → `purchase_request_summary/`

---

## Step 1 — Create via bench (developer mode required)

```bash
# Enable developer mode first
bench --site {site_name} set-config developer_mode 1
bench --site {site_name} clear-cache

# Then create the report from ERPNext UI:
# Awesome bar → "New Report" → fill details → Save
# Frappe auto-creates the folder and boilerplate files
```

Report settings in the UI:
- Report Type: **Script Report**
- Is Standard: **Yes**
- Module: your app module
- Reference DocType: the main DocType this report queries

## Step 2 — The Python file ({report_name}.py)

Always split into `execute`, `get_columns`, and `get_data` functions.

```python
# Copyright (c) 2024, Your Company
# License: MIT

import frappe
from frappe import _
from frappe.utils import getdate, nowdate


def execute(filters=None):
    """
    Main entry point. Must return (columns, data).
    Optionally also return: message, chart, report_summary, skip_total_row
    """
    filters = frappe._dict(filters or {})
    validate_filters(filters)

    columns = get_columns(filters)
    data = get_data(filters)
    chart = get_chart(data)
    summary = get_report_summary(data)

    return columns, data, None, chart, summary


def validate_filters(filters):
    if filters.from_date and filters.to_date:
        if getdate(filters.from_date) > getdate(filters.to_date):
            frappe.throw(_("From Date cannot be greater than To Date"))


def get_columns(filters):
    """
    Return list of column dicts. fieldtype controls formatting.
    Only return columns here if NOT defined in the Report UI.
    """
    return [
        {
            "label": _("Purchase Request"),
            "fieldname": "name",
            "fieldtype": "Link",
            "options": "Purchase Request",
            "width": 160,
        },
        {
            "label": _("Supplier"),
            "fieldname": "supplier",
            "fieldtype": "Link",
            "options": "Supplier",
            "width": 140,
        },
        {
            "label": _("Status"),
            "fieldname": "status",
            "fieldtype": "Data",
            "width": 100,
        },
        {
            "label": _("Required Date"),
            "fieldname": "required_date",
            "fieldtype": "Date",
            "width": 110,
        },
        {
            "label": _("Total Amount"),
            "fieldname": "total_amount",
            "fieldtype": "Currency",
            "options": "currency",       # links to a currency column for formatting
            "width": 130,
        },
        {
            "label": _("Currency"),
            "fieldname": "currency",
            "fieldtype": "Link",
            "options": "Currency",
            "width": 80,
            "hidden": 1,                 # hidden but needed for currency formatting
        },
    ]


def get_data(filters):
    """
    Build conditions safely using parameterised values — never f-string user input.
    Return list of dicts (as_dict=True).
    """
    conditions = get_conditions(filters)

    data = frappe.db.sql(
        """
        SELECT
            pr.name,
            pr.supplier,
            pr.status,
            pr.required_date,
            pr.total_amount,
            pr.currency
        FROM
            `tabPurchase Request` pr
        WHERE
            pr.docstatus < 2
            {conditions}
        ORDER BY
            pr.required_date ASC
        """.format(conditions=conditions),
        filters,
        as_dict=True,
    )

    return data


def get_conditions(filters):
    conditions = ""

    if filters.get("company"):
        conditions += " AND pr.company = %(company)s"

    if filters.get("supplier"):
        conditions += " AND pr.supplier = %(supplier)s"

    if filters.get("status"):
        conditions += " AND pr.status = %(status)s"

    if filters.get("from_date"):
        conditions += " AND pr.required_date >= %(from_date)s"

    if filters.get("to_date"):
        conditions += " AND pr.required_date <= %(to_date)s"

    return conditions


def get_chart(data):
    """Optional — returns chart config shown above the report table."""
    if not data:
        return None

    return {
        "data": {
            "labels": [d.get("supplier") for d in data[:10]],
            "datasets": [
                {
                    "name": _("Total Amount"),
                    "values": [d.get("total_amount", 0) for d in data[:10]],
                }
            ],
        },
        "type": "bar",          # bar | line | pie | donut | percentage
        "colors": ["#5e64ff"],
    }


def get_report_summary(data):
    """Optional — KPI tiles shown at the top of the report."""
    if not data:
        return None

    total = sum(d.get("total_amount", 0) for d in data)
    pending = sum(1 for d in data if d.get("status") == "Pending")

    return [
        {
            "value": len(data),
            "label": _("Total Requests"),
            "datatype": "Int",
            "indicator": "Blue",
        },
        {
            "value": pending,
            "label": _("Pending Approval"),
            "datatype": "Int",
            "indicator": "Orange" if pending > 0 else "Green",
        },
        {
            "value": total,
            "label": _("Total Value"),
            "datatype": "Currency",
            "currency": frappe.get_cached_value(
                "Company",
                frappe.defaults.get_user_default("Company"),
                "default_currency",
            ),
            "indicator": "Green",
        },
    ]
```

## Step 3 — The JavaScript filter file ({report_name}.js)

```javascript
// Copyright (c) 2024, Your Company
// License: MIT

frappe.query_reports["Purchase Request Summary"] = {
    filters: [
        {
            fieldname: "company",
            label: __("Company"),
            fieldtype: "Link",
            options: "Company",
            default: frappe.defaults.get_user_default("company"),
            reqd: 1,
        },
        {
            fieldname: "from_date",
            label: __("From Date"),
            fieldtype: "Date",
            default: frappe.datetime.month_start(),
            reqd: 1,
        },
        {
            fieldname: "to_date",
            label: __("To Date"),
            fieldtype: "Date",
            default: frappe.datetime.month_end(),
            reqd: 1,
        },
        {
            fieldname: "supplier",
            label: __("Supplier"),
            fieldtype: "Link",
            options: "Supplier",
        },
        {
            fieldname: "status",
            label: __("Status"),
            fieldtype: "Select",
            options: ["", "Draft", "Pending", "Approved", "Rejected", "Ordered"],
        },
    ],

    // Optional: format individual cells
    formatter: function (value, row, column, data, default_formatter) {
        value = default_formatter(value, row, column, data);

        if (column.fieldname === "status") {
            if (data.status === "Approved") {
                value = `<span style="color:green;font-weight:bold">${value}</span>`;
            } else if (data.status === "Rejected") {
                value = `<span style="color:red;font-weight:bold">${value}</span>`;
            } else if (data.status === "Pending") {
                value = `<span style="color:orange;font-weight:bold">${value}</span>`;
            }
        }

        return value;
    },
};
```

## Column fieldtype reference

| fieldtype | Use for | Notes |
|---|---|---|
| `Link` | DocType reference | set `options` to DocType name |
| `Data` | Plain text | — |
| `Currency` | Money values | set `options` to a currency column fieldname |
| `Float` | Decimal numbers | — |
| `Int` | Whole numbers | — |
| `Date` | Date values | — |
| `Datetime` | Date + time | — |
| `Percent` | 0–100 percentage | — |
| `Duration` | Time duration | — |
| `Check` | Boolean 0/1 | renders as checkbox |

## execute() return values — what each does

```python
return columns, data               # minimum required
return columns, data, None, chart  # with chart
return columns, data, None, chart, report_summary  # full
```

| Return position | Type | Purpose |
|---|---|---|
| 1 — `columns` | list of dicts | column definitions |
| 2 — `data` | list of dicts | row data |
| 3 — `message` | string or None | shown above the table |
| 4 — `chart` | dict or None | chart above the table |
| 5 — `report_summary` | list of dicts or None | KPI tiles at the top |
| 6 — `skip_total_row` | bool | pass `True` to hide the totals row |

## Advanced patterns

### Dynamic columns (pivot-style)

```python
def get_columns(filters, period_list):
    columns = [
        {"label": _("Supplier"), "fieldname": "supplier",
         "fieldtype": "Link", "options": "Supplier", "width": 160},
    ]
    for period in period_list:
        columns.append({
            "label": _(period["label"]),
            "fieldname": period["fieldname"],
            "fieldtype": "Currency",
            "width": 120,
        })
    return columns
```

### frappe.db.get_all instead of raw SQL

```python
# Preferred for simple queries — cleaner and safer
data = frappe.db.get_all(
    "Purchase Request",
    filters={
        "docstatus": ["<", 2],
        "company": filters.company,
        "required_date": ["between", [filters.from_date, filters.to_date]],
    },
    fields=["name", "supplier", "status", "required_date", "total_amount"],
    order_by="required_date asc",
)
```

### Multi-select filter in JS

```javascript
{
    fieldname: "supplier",
    label: __("Supplier"),
    fieldtype: "MultiSelectList",
    get_data: function (txt) {
        return frappe.db.get_link_options("Supplier", txt, {
            disabled: 0,
        });
    },
}
```

### Filter with on_change (auto date range)

```javascript
{
    fieldname: "period",
    label: __("Period"),
    fieldtype: "Select",
    options: ["Monthly", "Quarterly", "Yearly"],
    default: "Monthly",
    on_change: function () {
        const val = frappe.query_report.get_filter_value("period");
        if (val === "Monthly") {
            frappe.query_report.set_filter_value("from_date", frappe.datetime.month_start());
            frappe.query_report.set_filter_value("to_date", frappe.datetime.month_end());
        } else if (val === "Yearly") {
            frappe.query_report.set_filter_value("from_date", frappe.datetime.year_start());
            frappe.query_report.set_filter_value("to_date", frappe.datetime.year_end());
        }
        frappe.query_report.refresh();
    },
}
```

## Rules — always follow these

1. Always use `frappe._dict(filters or {})` at the top of execute()
2. Always use parameterised queries — `%(field)s` — never f-strings with user input
3. Use `as_dict=True` in `frappe.db.sql()` so rows are dicts not tuples
4. Validate date ranges with `frappe.throw(_("..."))` not Python raise
5. Use `_("string")` for every user-facing label for translation support
6. Keep `get_columns`, `get_data`, `get_conditions` as separate functions
7. Never hardcode company currency — use `frappe.get_cached_value`
8. Always run `bench migrate` after creating new report files
9. Test with: `bench --site {site} run-tests --module {app}.{module}.report.{report_name}`
10. Set `"Is Standard": "Yes"` so the report is saved to your app files

## Common errors and fixes

| Error | Cause | Fix |
|---|---|---|
| Report not found after save | Is Standard not set to Yes | Set Is Standard = Yes, save again |
| `execute() missing argument` | Wrong function signature | Use `def execute(filters=None):` |
| Column not showing | fieldname mismatch with data key | Make sure dict key matches `fieldname` |
| Currency not formatting | Missing `options` in currency column | Set `"options": "currency"` and return `currency` field in data |
| Filters not working | JS file name mismatch | JS file name must exactly match report name |
| `bench migrate` needed | New files not picked up | Always run `bench migrate` after creating report |
